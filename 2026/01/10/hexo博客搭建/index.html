hexo博客搭建

这篇就是给未来的自己看的：如果哪天把博客环境搞乱了，照着这篇一步一步重来就行

博客写作

  • 目前日常写博客就用obsidian，切换到blog工作区写就好了
  • 平板与电脑都有一个此工作区，写完同步就好了
  • 电脑做git提交，bat脚本运行就好了

一、整体目标

  1. 本地用 Hexo + Fluid 写博客、预览效果；
  2. 源码放在一个 Git 仓库（以下简称“源码仓库”）；
  3. 生成好的静态文件通过 GitHub Actions 自动推送到 roiiou/roiiou.github.io（以下简称“Pages 仓库”）；
  4. 最终访问地址是：https://roiiou.github.io。

  • site.name：导航栏左上角的名字；

  • theme_color.main：主题主色；

四、GitHub Pages 双仓库结构

这里采用的是 Hexo 官方推荐的双仓库方案：

  • 源码仓库：存 Hexo 源码（source/、_config*.yml、package.json 等）；
  • Pages 仓库：roiiou/roiiou.github.io，只存 public/ 里的静态文件，由 GitHub Pages 直接发布。

Pages 仓库设置：

  • 打开 roiiou/roiiou.github.io → Settings → Pages；
  • Source 设置为：Deploy from a branch；
  • Branch 选择：master，目录 / (root)。

要点：

  1. Action 使用 _config.yml 中的 theme: fluid + theme_dir，不再在脚本里 hexo config theme ...；
  2. 通过 npm 安装 hexo-theme-fluid，保证云端和本地主题版本一致；
  3. 生成后进入 public/，重新 git init，强制推送到 Pages 仓库的 master 分支；
  4. Pages 仓库的 GitHub Pages 设置为：master 分支根目录。

2. Personal Access Token（PAT）

为了让 Actions 能往 roiiou.github.io 推代码，需要一个 PAT：

  1. GitHub 右上角头像 → Settings → Developer settings → Personal access tokens；
  2. 生成一个 token（选 classic 或 fine-grained 都行，最少需要对 roiiou.github.io 有 contents: write 权限）；
  3. 复制 Token 值；
  4. 在 源码仓库 的 Settings → Secrets and variables → Actions 中添加：
    • Name：HEXO_DEPLOY_PAT
    • Value：刚才复制的 token

Action 中就是通过 ${{ secrets.HEXO_DEPLOY_PAT }} 来使用这个 token。
